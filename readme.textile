h4. to get to normal workflow of edit, revert, cache, commit, and push to remote.

# (edit files)
# checkout to revert changes, e.g. <pre>git checkout .</pre>
# insert changes to cache (also called index), e.g. <pre>git add src/Puzzles.cs</pre>
# remove changes out of cache (optional), e.g. <pre>git reset src/puzzles.java</pre>
# diff --cached before commit, e.g. <pre>git diff --cached</pre>
# commit changes, e.g. <pre>git commit -m 'added readme.textile.'</pre>
# push to remote git repo, e.g. <pre>git push git@github.com:0ishi/puzzles.git</pre>
# pull from remote git repo (optional), e.g. <pre>git pull</pre>
